# ikun 待机气泡与健康提醒设计

## 目标

调整 `ikun` 的气泡行为，并发布 `0.2.0`：

- 启动或切换到 `ikun` 时立即显示一次默认待机气泡；
- 待机气泡显示 5 秒后隐藏，不再常驻；
- 后续每隔随机 2～3 分钟再次显示 5 秒；
- 护眼提醒继续随机 30～50 分钟触发；
- 喝水提醒继续随机 60～90 分钟触发，并使用独立喝水文案；
- 每天 23:00 后首次空闲检测触发一次睡觉提醒；
- 点击、拖拽、桌面图标和健康提醒气泡优先于待机气泡。

## 行为

`ikun` 默认待机文案保持为：

> 中分头，背带裤，我是ikun你记住

首次进入 `ikun` 时立即显示，5 秒后气泡隐藏。后续刷新间隔在
120000～180000 毫秒之间均匀随机。计时到达时如果桌宠正在交互，则本轮不
覆盖交互气泡，重新安排下一次随机刷新。

护眼提醒保持现有文案和 8 秒时长：

> ikun们，看很久电脑了，要注意休息

喝水提醒使用独立文案和 8 秒时长：

> ikun们，练习久了也要记得喝水。

睡觉提醒每天本地时间 23:00 后触发一次，使用 8 秒时长：

> ikun们，夜深了，早点休息，明天再继续练习。

睡觉提醒日期写入本地存储，避免同一天重启后重复提醒；本地存储不可用时，
退化为当前运行周期内只提醒一次。

## 实现边界

- 纯行为规则放在 `src/pet-core/interaction.ts`，便于单元测试。
- 定时器、气泡显示与本地存储接入放在 `src/App.tsx`。
- 非 `ikun` 宠物保持现有待机气泡和健康提醒行为。
- 版本统一更新到 `0.2.0`，发布主程序和 NSIS 安装包。

## 验证

- 单元测试覆盖 2～3 分钟边界、5 秒显示时长、喝水文案和 23:00 每日一次。
- 全量运行 `npm test` 和 `npm run build`。
- 运行 `npm run tauri build`，确认生成 `愈心桌宠_0.2.0_x64-setup.exe`。
- 复制主程序和安装包到 `releases/`，并比较 SHA-256。
