# DS 独立台词包设计

日期：2026-06-19

状态：待用户审阅

## 1. 背景与目标

当前部分气泡文案仍由平台中的小橘逻辑提供，导致切换到 `ds` 后出现
“鱼！”、“吃到了。”以及其他属于 `xiaoju-cat` 的台词。

本次目标是为 `ds` 建立独立台词包：

- DS 的台词内容全部归属 `public/pets/ds/`。
- 平台只负责加载台词并按事件键读取。
- DS 不得在任何已覆盖事件中回退到小橘台词。
- 保持现有触发时机和动画机制，不新增提醒计时器或交互方式。

## 2. 人格与写作规则

DS 的核心性格是“可爱、有上进心”，主要通过温柔提醒用户一起做事、
一起进步来表达。

台词遵循以下规则：

- 主要使用“我们一起”的陪伴口吻，不强调“我”或“小鲸鱼”的自称。
- 提醒采用“具体照顾动作 + 回来继续努力”的结构。
- 可以少量使用“向前游”“摆摆尾巴”等鲸鱼特征。
- 不责备、不催促，不评价用户效率，也不制造亏欠感。
- 不使用“别偷懒”“怎么还没完成”等施压表达。
- 第一版每类事件只使用一句固定台词，不做随机台词池。

## 3. 事件与台词

| 事件键 | 触发场景 | 固定台词 |
| --- | --- | --- |
| `idle` | 待机时偶尔触发 | 慢慢来，我们一起向前游一点。 |
| `singleClick` | 单击 DS | 贴贴一下，我们一起加油吧。 |
| `doubleClick` | 双击 DS | 摆摆尾巴，我们一起向前冲一小步！ |
| `water` | 喝水提醒 | 去喝杯水吧，回来后我们再一起继续努力。 |
| `eyeCare` | 护眼提醒 | 看看远处，放松一下眼睛吧，休息好再一起继续。 |
| `meal` | 吃饭提醒 | 到吃饭时间啦，先补充能量，回来后我们继续努力。 |
| `sleep` | 睡觉提醒 | 今天已经很努力啦，早点休息吧，明天我们再一起出发。 |

双击触发的连续动画只显示 `doubleClick` 台词，不在动画中途切换为
“鱼！”或“吃到了。”。

## 4. 宠物包结构

在 DS 宠物包中增加独立台词文件：

```text
public/pets/ds/
  pet.json
  dialogues.json
  spritesheet.webp
  preview.png
```

`public/pets/ds/pet.json` 增加相对路径字段：

```json
{
  "dialoguesPath": "dialogues.json"
}
```

`dialogues.json` 使用固定事件键：

```json
{
  "idle": "慢慢来，我们一起向前游一点。",
  "singleClick": "贴贴一下，我们一起加油吧。",
  "doubleClick": "摆摆尾巴，我们一起向前冲一小步！",
  "water": "去喝杯水吧，回来后我们再一起继续努力。",
  "eyeCare": "看看远处，放松一下眼睛吧，休息好再一起继续。",
  "meal": "到吃饭时间啦，先补充能量，回来后我们继续努力。",
  "sleep": "今天已经很努力啦，早点休息吧，明天我们再一起出发。"
}
```

路径必须相对 DS 宠物包解析，不硬编码 `/pets/ds/` 或绝对 Windows 路径。

## 5. 平台职责

平台代码负责：

- 解析可选的 `dialoguesPath`。
- 使用现有宠物资产路径工具加载当前宠物的台词文件。
- 在切换宠物时同步切换当前台词包。
- 将待机、单击、双击、喝水、护眼、吃饭和睡觉事件映射到对应键。
- 双击动画序列期间保持同一句 DS 台词。
- 在文件或事件键缺失时隐藏该次气泡并记录警告。

宠物包负责：

- DS 的人格、具体措辞和事件台词。
- 后续可能增加的同事件随机变体。

平台不得：

- 为 DS 使用 `getTimedDefaultBubble()` 中的小橘文案。
- 将 DS 的台词硬编码为 `activePetId === "ds"` 分支。
- 在 DS 台词加载失败时回退到 `xiaoju-cat` 台词。

## 6. 运行流程

1. 平台加载当前宠物的 `pet.json`。
2. 如果存在 `dialoguesPath`，按宠物包相对路径加载台词文件。
3. 当前事件发生时，平台使用事件键读取当前宠物台词。
4. 找到非空台词时显示气泡，并沿用现有气泡持续时间。
5. 未找到台词或文件加载失败时，本次不显示气泡，并记录可诊断警告。
6. 切换宠物后，旧宠物台词不得继续保留或参与回退。

吃饭和睡觉仍沿用现有基于时间段的提醒机制，只替换其文案来源。
喝水、护眼、待机、单击和双击同样保持现有触发频率与动画映射。

## 7. 兼容与异常处理

`dialoguesPath` 是可选能力。未声明该字段的现有宠物继续沿用当前行为，
本次不强制迁移小橘和 ikun。

DS 的失败策略更严格：

- `dialogues.json` 加载失败：DS 不显示相关气泡并记录警告。
- 单个事件键缺失或值为空：该事件不显示气泡并记录警告。
- JSON 格式无效：视为加载失败，不使用其他宠物台词补齐。
- 快速切换宠物时：过期加载结果不得覆盖当前宠物的台词包。

动画和交互不能因台词加载失败而停止。

## 8. 测试与验收

自动化测试覆盖：

- `dialoguesPath` 使用宠物包相对路径解析。
- DS 七类事件都映射到已确认的固定台词。
- DS 双击序列始终使用 `doubleClick` 台词。
- DS 台词缺失或加载失败时返回空结果，不回退到小橘。
- 宠物切换后不会继续使用上一个宠物的台词。
- DS 的输出中不出现“喵”、“小橘”、“鱼！”或“吃到了。”。
- 未声明 `dialoguesPath` 的现有宠物行为保持兼容。

实现完成后运行：

- 台词加载与交互相关的定向测试。
- `npm test`。
- `npm run build`。

体验验收：

- 七类触发显示对应 DS 台词。
- 双击动作全过程没有小橘台词闪现。
- 提醒语气温柔、具体，不包含责备或施压。
- 少量鲸鱼特征自然出现，不影响“陪你做事、一起进步”的主基调。

## 9. 非目标

- 不调整气泡样式、位置或持续时间。
- 不增加新的提醒类型、计时器或设置项。
- 不为每类事件增加随机台词。
- 不在本次迁移小橘或 ikun 的全部台词。
- 不修改 DS 动画图集、声音或其他宠物资源。
