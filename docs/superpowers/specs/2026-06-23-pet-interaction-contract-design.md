# 桌宠基础交互契约设计

## 1. 背景与目标

当前平台已经支持待机、点击、拖拽、健康提醒和桌面图标探测，但这些行为主要集中在 `src/App.tsx` 和 `src/pet-core/interaction.ts` 的 pet-id 分支中。各行为共用返回待机计时器，平台只记录当前动画名，不记录交互来源、优先级或生命周期，因此会出现：

- 拖拽窗口跟随指针，但宠物朝向不跟随水平拖拽方向。
- 点击、提醒、图标交互和待机小动作会互相覆盖收尾计时器。
- 右键单击只播放待机，更新检查仍由左键三击、托盘菜单和启动自动检查触发。
- 发现新版本后直接打开 Release 页面，没有用户确认。
- 四种健康提醒并非所有 pet 都完整支持。
- 图标互动存在错误回退、卡在 `iconHug`、方向定位不一致等问题。
- 部分动作尺寸跳变明显，小橘部分动作还会读取透明空帧。

本设计建立平台级统一交互契约，使所有 pet 以相同事件模型运行，同时将动作内容、素材、帧数、缩放和可选能力保留在各自 pet 包内。

## 2. 范围

### 2.1 本轮包含

- 所有 pet 的待机、单击、双击和左右拖拽。
- 右键双击检查更新及确认窗口。
- 护眼、喝水、吃饭、睡觉四种非主动提醒。
- 桌面应用图标非主动交互。
- 待机小动作和小橘专属悬停。
- 统一交互优先级、抢占、延迟和返回待机规则。
- 动作有效帧、视觉尺寸、基线、方向和进入/退出衔接。
- Windows 桌面图标识别能力的显式启停。

### 2.2 本轮不包含

- 应用内直接下载或安装新版本；确认后暂时只打开 GitHub Release 页面。
- 为 ikun 或蒜鸟设计新的桌面图标交互动作。
- macOS 桌面图标枚举实现。
- 为每个缺少健康提醒素材的 pet 立即制作新动画。
- 重做所有 pet 图集；只修复本轮验收发现的必要素材问题。

## 3. 已确认的产品规则

### 3.1 基础事件

所有 pet 必须支持：

- `idle`
- `singleClick`
- `doubleClick`
- `drag`
- `eyeCare`
- `water`
- `meal`
- `sleep`

可选能力：

- `hover`：默认关闭，本轮仅小橘开启。
- `desktopIcon`：本轮仅小橘和 ds 开启；ikun 和蒜鸟显式关闭。
- `idleQuirk`：可按 pet 配置动作列表；只能从待机触发。

### 3.2 交互优先级

从高到低：

1. 更新确认窗口、拖拽、单击、双击。
2. 护眼、喝水、吃饭、睡觉提醒。
3. 桌面图标交互。
4. 待机小动作。
5. 基础待机。

高优先级事件可以抢占低优先级事件。低优先级事件不得抢占高优先级事件，可进入一个最多保留一项的延迟队列。健康提醒延迟后最多补发一次，不连续追补已经错过的多个提醒。

### 3.3 更新检查

- 删除启动 3 秒后的自动更新检查。
- 删除左键三击检查更新。
- 保留托盘菜单“检查更新”。
- 新增宠物区域右键双击检查更新。
- 普通右键单击只阻止系统菜单，不播放动作、不检查更新。
- 检查期间防止并发重复请求。
- 当前版本从应用构建版本读取，禁止硬编码。
- 版本使用语义版本比较，只有远端版本更高时才判定有更新。
- 发现新版本后显示应用内确认窗口，展示当前版本、最新版本和可用更新说明。
- 用户选择“前往下载”后才打开 GitHub Release 页面。
- 用户取消后关闭窗口并恢复之前的待机状态。
- 检查失败或已是最新版时给出明确反馈，但不打开网页。

## 4. 平台交互状态模型

新增独立于 Pixi 动画名的运行时状态：

```ts
type PetInteractionKind =
  | "idle"
  | "idleQuirk"
  | "hover"
  | "desktopIcon"
  | "careReminder"
  | "singleClick"
  | "doubleClick"
  | "drag"
  | "updatePrompt";

type PetFacing = "left" | "right";
type PetInteractionPhase = "enter" | "loop" | "exit";

type ActivePetInteraction = {
  token: number;
  kind: PetInteractionKind;
  priority: number;
  phase: PetInteractionPhase;
  facing: PetFacing;
  interruptible: boolean;
};
```

### 4.1 令牌

每次开始新交互时递增 `token`。所有超时回调、Pixi `onComplete` 和异步图标定位回调都必须捕获启动时令牌；令牌不再匹配时，回调不得修改动画、气泡、鼠标穿透或当前状态。

这会替代“所有动作共享一个无归属的 `returnToIdleTimer`”的行为，防止旧动作稍后把新动作强制切回待机。

### 4.2 抢占与等待

- 直接交互立即抢占提醒、图标互动和待机动作。
- 健康提醒只在没有直接交互时播放。
- 图标互动只允许从基础待机触发。
- 待机小动作只允许从基础待机触发。
- 低优先级事件被阻止时，只保留最新的一项可补发事件。
- 切换 pet、打开平台或销毁 Pixi 应用时，取消活动令牌、计时器、鼠标穿透和等待事件。

## 5. Pet 包交互声明

共享事件识别、调度和渲染逻辑属于 `src/`。Pet 特定动作、文案、帧数、缩放、偏移、方向素材和能力启停属于 `public/pets/<pet-id>/pet.json`。

Manifest 扩展为以下结构；实施时字段语义保持一致，不再使用 pet-id 分支推断能力：

```json
{
  "interactions": {
    "idle": {
      "animation": "idle"
    },
    "singleClick": {
      "animation": "tickle",
      "durationMs": 1200
    },
    "doubleClick": {
      "sequence": ["fishChase", "fishEat"]
    },
    "drag": {
      "right": { "animation": "drag" },
      "left": { "animation": "dragLeft" },
      "directionMode": "rows"
    },
    "hover": {
      "enabled": false
    },
    "desktopIcon": {
      "enabled": false
    },
    "reminders": {
      "eyeCare": { "animation": "idle" },
      "water": { "animation": "idle" },
      "meal": { "animation": "idle" },
      "sleep": { "animation": "idle" }
    }
  },
  "animations": {
    "tickle": {
      "row": 3,
      "frames": 4,
      "speed": 0.16,
      "loop": false,
      "scale": 0.84,
      "offsetX": 0,
      "offsetY": 0
    }
  }
}
```

该结构必须满足以下约束：

- 实际有效帧数必须由 pet 包明确声明。
- 不允许通过缺少配置自动推断“支持”可选能力。
- 不允许将缺少 `desktopIcon` 素材的 pet 回退到普通待机并声称正在互动。
- 所有资源路径仍必须相对 pet 包目录。

## 6. 拖拽方向

### 6.1 根因

当前拖拽只计算窗口位置，没有计算或保存水平拖拽方向，运行时始终播放一个固定动作行。

### 6.2 方向判定

- 使用连续指针样本累计相对最近方向锚点的水平位移。
- 累计水平位移达到 8 个逻辑像素时更新朝向并重置方向锚点。
- 8 像素滞后阈值避免在小幅左右抖动时连续翻转。
- 主要为垂直拖拽时保持最近有效朝向。
- 新一次拖拽开始时沿用 pet 最近朝向；首次启动使用 manifest 默认朝向。

方向阈值固定为 8 个逻辑像素，并通过单元测试覆盖高 DPI 下的坐标换算。行为要求是：明显左右拖拽及时转向，手部轻微抖动不触发转向。

### 6.3 各 pet 的方向策略

| Pet | 策略 |
| --- | --- |
| 小橘 | 使用图集第 1 行向右和第 2 行向左。 |
| ds | 使用第 1 行向右和第 2 行向左。 |
| 蒜鸟 | 使用第 1 行向右和第 2 行向左，保留起飞、循环、落地阶段。 |
| ikun | 当前没有合适的独立左向拖拽行；镜像后撤步动作。若后续素材含文字或必须保持左右手道具关系，可在 manifest 禁止镜像并补独立素材。 |

## 7. 健康提醒

### 7.1 所有 pet 必须覆盖四类提醒

- 护眼
- 喝水
- 吃饭
- 睡觉

有专属动作时播放专属动作。没有合适动作时，使用该 pet 的稳定待机动画和专属气泡，不用吃鱼、投球等无关动作冒充提醒。

### 7.2 时间与去重

- 护眼：保留随机 30–50 分钟间隔。
- 喝水：保留随机 60–90 分钟间隔。
- 早餐：07:00–09:00。
- 午餐：11:00–13:00。
- 晚餐：18:00–20:00。
- 睡眠：23:00–次日 06:00。
- 饭点和睡眠提醒按日期与时段持久化去重，应用重启后不重复。
- 切换 pet 不应造成同一平台提醒时段重复触发。

### 7.3 文案

有 `dialogues.json` 的 pet 从包内读取 `eyeCare`、`water`、`meal`、`sleep`。缺少对应文案时记录警告，并使用平台中性文案，不使用“小橘”“喵”等其他 pet 的人格文案。

## 8. 桌面图标交互

### 8.1 共享识别流程

- Windows 下每 1.2 秒探测一次桌面状态和 Explorer 图标位置。
- 宠物锚点距离目标图标约 38px 内时开始计时。
- 对同一图标持续停留 5 秒后触发。
- 触发后使用 15 秒冷却。
- 只允许从基础待机触发，不得打断直接交互、健康提醒或更新窗口。
- macOS 和浏览器预览显式视为能力不可用，不尝试假交互。

### 8.2 能力范围

| Pet | 状态 | 行为 |
| --- | --- | --- |
| 小橘 | 启用 | 使用独立拥抱图标动作。需要清理现有素材中的彩色块、箱体和边缘伪影，并保证结束后恢复待机和鼠标事件。 |
| ds | 启用 | 使用探头动作。根据图标位于宠物左侧或右侧选择对应窗口定位，避免跨越图标。 |
| ikun | 禁用 | 本轮不触发铁山靠或独立图标素材，等待后续单独讨论。 |
| 蒜鸟 | 禁用 | 留空等待后续补素材，不允许回退到待机冒充图标互动。 |

### 8.3 清理保证

无论成功、取消、被抢占或定位失败，都必须：

- 恢复 `setIgnoreCursorEvents(false)`。
- 清除图标锁定状态。
- 使当前交互令牌失效。
- 回到合适的待机状态。
- 重新允许后续图标探测。

## 9. 悬停与待机小动作

- 小橘保留 800ms 悬停后的鱼动作序列。
- ds 不再自动触发悬停动作，避免超出统一基线。
- ikun 和蒜鸟保持无悬停。
- 点击结束后不立即重新武装悬停，避免单击后再次自动触发。
- 拖拽结束回到待机后可以重新武装小橘悬停。
- 待机小动作只能在基础待机状态触发，并服从统一优先级。

## 10. 动画与视觉验收

### 10.1 通用标准

- 角色躯干视觉尺寸在普通动作间变化约不超过 12%；蹲下、睡姿、跳跃和飞行的合理姿态变化不计为体型错误。
- 地面动作脚底保持稳定基线；跳跃和飞行通过 `enter/loop/exit` 阶段表达垂直位移。
- 任何运行时帧不得完全透明。
- 使用每动作缩放和可选 `offsetX`、`offsetY` 校正，不根据整个 192×208 透明单元格判断角色尺寸。
- 优先使用真实左右素材；没有素材时才允许水平镜像。
- 不使用透明度交叉淡化，避免半透明毛边和双影。
- 动作进入时从当前姿势选择最接近的起始帧。
- 动作退出优先使用独立退出帧；没有退出帧时短暂停留末帧，再切到最接近的待机帧。

### 10.2 已知 pet 修复项

#### 小橘

- 单击行只播放 4 个有效帧，不读取后 4 个透明单元格。
- 双击第一段只播放 6 个有效帧，不读取后 2 个透明单元格。
- 清理 `icon-hug.webp` 的彩色块、箱体和边缘伪影。
- 调整睡姿、直立和抱鱼动作的缩放，使角色身份比例稳定。
- 接入左右拖拽行。

#### ikun

- 双击跳跃增加合理落地到待机衔接。
- 拖拽根据方向镜像。
- 桌面图标交互显式禁用，本轮不修图标素材。

#### ds

- 保持现有较稳定的动作尺度。
- 校正跳跃动作中非意图性的基线变化。
- 接入左右拖拽行。
- 图标探头根据图标方向定位。
- 关闭自动悬停。

#### 蒜鸟

- 保留已有起飞、飞行循环和落地阶段。
- 接入左右拖拽行。
- 单击和双击结束时选择接近待机的返回姿势。
- 桌面图标交互显式禁用。

## 11. 错误处理

- Manifest 交互配置无效时记录 pet id、事件名和字段错误，并禁用该事件；不得加载越界帧。
- 资源加载失败时回到基础待机并保持其他交互可用。
- 更新请求失败时关闭检查中状态并显示错误反馈。
- Release URL 打开失败时保留确认窗口或显示可重试反馈。
- 桌面图标读取失败只禁用当前探测周期，不影响点击、拖拽和提醒。
- 浏览器预览中所有 Tauri 调用继续使用运行时检测安全跳过。

## 12. 测试与验证

### 12.1 单元测试

- Manifest 交互声明解析和默认禁用规则。
- 四个 pet 的基础事件覆盖。
- 小橘悬停启用，其他 pet 悬停禁用。
- 小橘和 ds 图标交互启用；ikun 和蒜鸟禁用。
- 拖拽方向阈值、滞后和垂直移动保持朝向。
- 左右行选择和 ikun 镜像策略。
- 交互优先级、抢占、等待和令牌失效。
- 四种提醒映射、待机降级和持久化去重。
- 右键单击无动作、右键双击检查更新。
- 语义版本比较、更新确认、取消和打开 Release。
- 动作有效帧数、透明帧检测、缩放后边界和基线范围。

### 12.2 集成与视觉验证

每个 pet 分别验证：

- 待机 → 单击 → 待机。
- 待机 → 双击 → 待机。
- 向左拖拽 → 落地 → 待机。
- 向右拖拽 → 落地 → 待机。
- 护眼、喝水、吃饭、睡觉提醒。
- 高优先级操作抢占低优先级动作后，旧计时器不再生效。
- 小橘和 ds 的桌面图标互动结束后恢复待机。
- ikun 和蒜鸟靠近图标后不触发动作。

平台验证：

- Windows 验证真实 Explorer 图标识别。
- 浏览器预览验证 Tauri API 安全跳过。
- macOS 验证图标能力显式关闭。
- 更新确认窗口验证键盘焦点、取消和确认路径。

此外需要运行图集透明边界检查和浏览器/Tauri 视觉回归，不以单元测试通过代替实际动作验收。

## 13. 实施顺序

1. 建立 manifest 交互声明和解析测试。
2. 建立统一交互状态、优先级、令牌和等待队列。
3. 接入单击、双击、待机和悬停。
4. 接入拖拽方向与各 pet 左右策略。
5. 接入四种健康提醒及持久化去重。
6. 接入图标能力启停、方向定位和清理保证。
7. 实现右键双击更新检查与确认窗口。
8. 修复有效帧、素材伪影、缩放、偏移和动作过渡。
9. 完成自动测试、四 pet 视觉回归和平台验证。
