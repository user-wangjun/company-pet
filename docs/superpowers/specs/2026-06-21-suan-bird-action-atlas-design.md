# 蒜鸟动作图集设计

## 目标

根据用户提供的《蒜鸟动作设定图》，替换当前蒜鸟占位式动作图集，使待机、单击、双击以及喝水、护眼、吃饭、睡觉提醒具有明确且连贯的角色动作。

## 约束

- 蒜鸟素材继续完全位于 `public/pets/suan-bird/`。
- 保持现有 `1536x1872` WebP 图集、`192x208` 单元格、8 列 9 行契约。
- 不改变其他桌宠的动作、帧数或提醒行为。
- 设定图仅作为角色和分镜参考；最终帧不包含文字、编号、白底、阴影或游离特效。
- 动作帧保持统一角色比例、脚底基线、蒜头造型、绿色领巾和奶油色羽毛。

## 动作行映射

| 行 | 运行时动作 | 蒜鸟表现 |
| --- | --- | --- |
| 0 | `idle` | 轻微呼吸、眨眼、左右小幅摇摆 |
| 1 | `drag` | 朝右侧身扑翼 |
| 2 | 保留方向行 | 朝左侧身扑翼 |
| 3 | `tickle` | 单击后趴下、弹起、开心回应 |
| 4 | `crouchAlert` | 举起水杯并喝水 |
| 5 | `hugFish` | 戴上绿色眼镜并做护眼提醒 |
| 6 | `gnawFish` | 端碗、拿饭团并开心吃饭 |
| 7 | `fishChase` | 双击后钻入绿色条纹被窝 |
| 8 | `fishEat` | 戴睡帽、裹被熟睡 |

双击继续使用现有 `fishChase -> fishEat` 序列。睡觉提醒直接播放 `fishEat`，避免新增图集行或扩大运行时纹理契约。

## 运行时接入

将健康提醒动作查询扩展为带事件类型的接口：

```ts
type PetCareReminderKind = "water" | "eyeCare" | "meal" | "sleep";

getPetCareReminderAction(petId, kind)
```

蒜鸟按事件返回上表中的专属动作；已有桌宠维持当前统一提醒动作。喝水和护眼沿用现有随机提醒计时器。吃饭和睡觉使用现有时段规则，每个日期和时段最多主动播放一次，避免桌宠探测循环重复触发。

## 素材流程

以用户动作设定图和现有蒜鸟三视图为共同参考，生成无文字的动作条或关键姿势；使用洋红色纯色背景做抠图，再统一缩放、脚底基线和透明像素。最终输出：

- `spritesheet.webp`
- `preview.png`
- `qa/contact-sheet.png`
- `qa/previews/*.gif`
- `qa/validation.json`
- 保留原始动作参考图和生成源文件在蒜鸟包内

## 测试与验收

- 单元测试覆盖蒜鸟四种提醒的动作映射，以及其他桌宠行为不变。
- 定时提醒测试覆盖餐时、睡眠时段和同一时段去重。
- 图集验证尺寸、帧数、透明像素和空闲单元格。
- 视觉检查确认角色一致、无裁切、无白底、无明显尺寸跳动。
- 运行 `npm test -- src/pet-core/interaction.test.ts`、`npm test -- src/pet-core/petAssets.test.ts`、`npm test` 和 `npm run build`。
